# 交易指南

Quanta 交易操作完整指南。

## 交易模式

| 模式         | 市场数据 | 执行 | 风险 | 适用场景 |
| ------------ | -------- | ---- | ---- | -------- |
| **simulate** | 模拟     | 模拟 | 无   | 学习     |
| **paper**    | 真实     | 模拟 | 无   | 策略验证 |
| **live**     | 真实     | 真实 | 高   | 生产     |

### 模拟模式

**用途**: 使用合成数据学习和初始测试

```bash
quanta trade start --env simulate --coins BTC,ETH,SOL
```

**特性:**

- 使用内部模拟交易所（`simulator`）
- 完全无风险环境
- 可使用 Mock AI 或 Real AI
- 不涉及真实资金
- 无外部 API 依赖

**适用场景:**

- 学习系统工作原理
- 测试新功能
- 理解算法
- 初始 AI 提示词测试

### 纸面交易模式

**用途**: 用真实市场条件验证策略，无财务风险

```bash
quanta trade start --env paper --coins BTC,ETH,SOL
```

**特性:**

- 获取**真实市场数据**（来自实际交易所）
- 模拟订单执行和持仓管理
- 基于实时价格变动跟踪真实 P&L
- 使用真实市场波动、趋势和模式
- **API 密钥可选**（无需凭据即可获取公共市场数据）

**适用场景:**

- 用真实市场条件验证策略
- 在实时数据上测试 AI 性能
- 了解策略在波动市场中的表现
- 在实际交易前完善风险参数

### 实盘模式

**用途**: 使用实际资金执行真实交易

```bash
quanta trade start --env live --coins BTC
```

**要求:**

- 具有交易权限的有效 API 密钥
- 正确的风险管理配置
- 在模拟和纸面交易中充分测试

**⚠️ 重要警告:**

- **真实资金面临风险** - 损失是永久的
- 始终先在模拟/纸面模式测试
- 从小仓位大小开始
- 主动监控持仓
- 了解交易所费用结构
- 注意滑点和执行质量
- 知道如何立即停止交易（`Ctrl+C`）

## 交易生命周期

### 1. 开始交易

```bash
# 模拟模式
quanta trade start --env simulate --coins BTC,ETH,SOL

# 纸面交易模式
quanta trade start --env paper --coins BTC,ETH,SOL

# 实盘交易
quanta trade start --env live --coins BTC
```

### 2. 监控输出

```bash
# 实时查看详细输出
quanta log view --follow

# 使用特定筛选器查看
quanta log view --follow --context Workflow --level info
```

### 3. 控制交易

使用 `Ctrl+C` 停止正在运行的交易进程。

### 4. 查看结果

```bash
# 运行带增强报告的回测
quanta trade backtest --start 2024-01-01 --end 2024-12-31 --coins BTC,ETH --initial-balance 10000
```

回测报告包括:

- 信号统计（生成、接受、拒绝）
- 性能摘要（收益、夏普比率、回撤）
- 交易统计（胜率、盈亏比、最佳/最差交易）
- 风险指标（波动性、VaR、最大回撤）
- 权益曲线分析

### 工具和定价

- **OKX 工具**: Quanta 对永续合约使用 `BASE/USDT:USDT`（例如 `ETH/USDT:USDT`）
- **入场定价**: 执行参考实时中间价（最佳买卖价平均值），而非 K 线收盘价
- **显示的风险敞口**: 绝对持仓价值之和（大小 × 标记价格），不含杠杆乘数

### 市场类型

| 类型          | 杠杆   | 资金费率 | 做空 | 风险水平 |
| ------------- | ------ | -------- | ---- | -------- |
| **现货**      | 仅 1x  | 无       | 否   | 较低     |
| **合约/永续** | 3x-10x | 是       | 是   | 较高     |

**推荐配置:**

- **现货**: 杠杆 [1,1]，止损 3–7%，最大风险 3–5%，最大持仓 6–10
- **合约/永续**: 杠杆 [3,10]，止损 1–2%，最大风险 1–2%，最大持仓 1–4

## 风险管理

Quanta 自动实施风险控制:

### 仓位大小

- **每笔交易最大风险**: 账户权益的 5%
- **资本配置**: 每个持仓可用交易资本的 30%
- **最小仓位**: 权益的 1% 或 $200（取较大值）
- **资本储备**: 保留 40% 作为额外持仓储备

### 止损

- **默认**: 5%（可配置）
- **类型**: 基于百分比、基于 ATR 或固定金额
- **设置**: 做多设在入场价下方，做空设在入场价上方

### 止盈

- **默认**: 6%（止损的 2 倍）
- **策略**: 固定、跟踪或多级

### 组合限制

- **最大持仓**: 6 个并发持仓（可配置）
- **总风险上限**: 账户权益的 30%
- **置信度阈值**: 0.55（55%）最小信号置信度

### 杠杆

- **范围**: 衍生品 5x 到 40x，现货 1x
- **安全**: 根据市场类型自动限制

详见 [核心概念](concepts.md#风险管理)。

## 盈亏（P&L）

### 理解 P&L

**未实现盈亏**: 持仓中的盈亏（尚未平仓）

```typescript
// 做多持仓
未实现盈亏 = (当前价格 - 开仓价格) × 持仓大小

// 做空持仓
未实现盈亏 = (开仓价格 - 当前价格) × 持仓大小
```

**已实现盈亏**: 平仓持仓的盈亏（实际现金获得/损失）

```typescript
已实现盈亏 = 平仓时计算;
账户余额 += 已实现盈亏;
```

### 账户价值

```typescript
余额 = 初始资金 + 所有已实现盈亏
权益 = 余额 + 未实现盈亏（来自持仓）
可用保证金 = 权益 - 已用保证金
```

### 杠杆对 P&L 的影响

使用 10x 杠杆:

- 仓位价值: $10,000（10 BTC @ $1,000）
- 所需保证金: $1,000（仓位的 10%）
- 1% 价格变动 → $100 P&L（保证金 ROI 10%）

### 交易跟踪

所有完成的交易记录:

- 开仓/平仓时间和价格
- 持仓大小和方向（做多/做空）
- 已实现盈亏（绝对值和百分比）
- 持有期
- 平仓原因（信号/止损/止盈）

## 最佳实践

### ✅ 应该做

- 始终先在模拟或纸面交易模式测试
- 推荐纸面交易用于策略验证（真实数据）
- 从小仓位大小开始
- 定期监控持仓
- 使用止损
- 跨币种分散
- 理解风险管理参数

### ❌ 不应该做

- 不要使用无法承受损失的资金交易
- 不要禁用风险控制
- 不要忽略警告
- 不要过度杠杆
- 不要跳过测试阶段

```bash

quanta config show

# 验证设置
quanta config validate

# 检查状态（通过日志）
quanta log view --context TradeStart --lines 100
```

### API 错误

```bash
# 测试交易所连接
quanta test exchange --exchange simulator --coin BTC

# 测试 AI
quanta test ai --type mock --coin BTC

# 检查 API 密钥
quanta config show | grep -i api
```

### 性能问题

```bash
# 查看详细日志
quanta log view --follow --context Workflow

# 检查错误
quanta log view --level error

# 导出日志进行分析
quanta log export --output logs.json --format json
```
